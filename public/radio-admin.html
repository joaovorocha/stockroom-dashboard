Manager Quick Setup (simple)
Use this if you just need the system running. The defaults are already set for PMR446 Channel 2.
1) Keep Mode on Manual (Locked) for a stable channel.
2) Use PPM 25 and Gain 35 (recommended start values).
3) If words get cut off at the start, increase Pre‑roll to 300–400 ms.
Configuration

Friendly tuning controls. Save & Restart applies changes to the running service.

Manual = lock one channel. Automatic = scan channels. Use Manual for stable speech capture.
Switches the tuner to another saved channel. Applies live (no restart).
Shown next to transcriptions (e.g. “Stockroom (Ch 2)”).
Set the center frequency (example: 446.01875M). Common PMR446 (Motorola/Talkabout) channels are often: Ch1 446.00625, Ch2 446.01875, Ch3 446.03125, Ch4 446.04375, Ch5 446.05625, Ch6 446.06875, Ch7 446.08125, Ch8 446.09375, Ch9 446.10625, Ch10 446.11875, Ch11 446.13125, Ch12 446.14375, Ch13 446.15625, Ch14 446.16875, Ch15 446.18125, Ch16 446.19375. (Your store radios may be programmed differently—use the exact frequency from the radio programming.)
Frequency correction for the RTL-SDR. If the audio sounds “off” or the signal is shifted, try small values like -10 to +10.
Tuner gain. Use 0 for auto. Increase slowly if the signal is weak; too high can distort.
Higher = less noise triggers (fewer false transcriptions) but may miss quiet speech. Typical range: 0.02–0.06.
How long to keep recording after speech drops below the threshold. Higher = fewer cut-offs, but longer segments (more CPU).
Lower = more sensitive (opens on weaker signals). Try 8–12 if audio never opens.
Adds a small buffer before voice starts so the first word isn’t clipped. Try 250–400 ms.
Advanced (ASR model + device)
Bigger models are more accurate but use more CPU. For lowest CPU, keep “tiny”.
CPU works everywhere. CUDA requires an NVIDIA GPU + drivers on the server.
For CPU, “int8” is typically the fastest and lowest-memory option.
