# MCP Doors Configuration Guide

## Overview

MCP (Model Context Protocol) Doors provide AI agents with secure, structured access to your Stockroom Dashboard systems. Each "door" is an MCP server that exposes specific tools for different aspects of your operations.

## Architecture

```
┌─────────────────┐    ┌──────────────────┐
│   AI Agent      │────│  MCP Client      │
│   (Claude, etc) │    │  Configuration   │
└─────────────────┘    └──────────────────┘
                              │
                    ┌─────────┼─────────┐
                    │         │         │
            ┌───────▼───┐ ┌───▼───┐ ┌───▼───┐
            │Inventory  │ │Shipments│ │ Radio │
            │  Server   │ │ Server │ │Server │
            └───────────┘ └────────┘ └───────┘
                    │         │         │
            ┌───────▼───┐ ┌───▼───┐ ┌───▼───┐
            │Stockroom  │ │Stockroom│ │Radio │
            │Database   │ │Database│ │Service│
            └───────────┘ └────────┘ └───────┘
```

## Available MCP Doors

### 1. Stockroom Inventory Server (`inventory-server.js`)
Provides tools for inventory management and queries.

**Tools:**
- `get_inventory_status` - Get inventory statistics and counts
- `search_inventory` - Search for items by name, SKU, or description
- `get_item_details` - Get detailed information about specific items
- `update_inventory` - Update item information and quantities

### 2. Stockroom Shipments Server (`shipments-server.js`)
Provides tools for shipping and package management.

**Tools:**
- `get_shipments` - List shipments with filtering options
- `get_shipment_details` - Get detailed shipment information
- `create_shipment` - Create new shipment records
- `update_shipment_status` - Update shipment status and tracking
- `get_shipping_stats` - Get shipping metrics and statistics

### 3. Stockroom Radio Server (`radio-server.py`)
Provides tools for radio communication monitoring.

**Tools:**
- `get_radio_status` - Get current radio monitoring status
- `get_radio_transcripts` - Retrieve recent radio transcripts
- `monitor_frequency` - Start monitoring specific frequencies
- `get_radio_alerts` - Get active radio alerts and notifications

## Configuration

### 1. Environment Variables

Each MCP server requires specific environment variables:

```bash
# For all servers
STOCKROOM_DATA_DIR=/var/lib/stockroom-dashboard/data
STOCKROOM_FILES_DIR=/var/lib/stockroom-dashboard/files

# For radio server
PYTHONPATH=/var/www/stockroom-dashboard
```

### 2. Client Configuration

Create a client configuration file (e.g., `mcp-client-config.json`):

```json
{
  "mcpServers": {
    "stockroom-inventory": {
      "command": "node",
      "args": ["mcp-servers/inventory-server.js"],
      "env": {
        "STOCKROOM_DATA_DIR": "/var/lib/stockroom-dashboard/data",
        "STOCKROOM_FILES_DIR": "/var/lib/stockroom-dashboard/files"
      }
    },
    "stockroom-shipments": {
      "command": "node",
      "args": ["mcp-servers/shipments-server.js"],
      "env": {
        "STOCKROOM_DATA_DIR": "/var/lib/stockroom-dashboard/data"
      }
    },
    "stockroom-radio": {
      "command": "python",
      "args": ["mcp-servers/radio-server.py"],
      "env": {
        "PYTHONPATH": "/var/www/stockroom-dashboard"
      }
    }
  }
}
```

### 3. AI Client Integration

#### For Claude Desktop:
Add to your `claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "stockroom": {
      "command": "node",
      "args": ["/var/www/stockroom-dashboard/mcp-servers/inventory-server.js"],
      "env": {
        "STOCKROOM_DATA_DIR": "/var/lib/stockroom-dashboard/data"

    }
  }

```

#### For VS Code Copilot:
Configure in VS Code settings:

```json

```

## Setup Instructions

1. **Run the setup script:**
   ```bash
   ./setup-mcp-doors.sh
   ```

2. **Install dependencies:**
   ```bash
   npm install @modelcontextprotocol/sdk
   python -m pip install mcp
   ```

3. **Configure your AI client** using the client configuration file generated by the setup script.

4. **Test the servers:**
   ```bash
   # Test inventory server
   node mcp-servers/inventory-server.js

   # Test shipments server
   node mcp-servers/shipments-server.js

   # Test radio server
   python mcp-servers/radio-server.py
   ```

## Security Considerations

- MCP servers run with the same permissions as your main application
- Data access is controlled by your existing authentication and authorization
- Consider network isolation if deploying in production
- Monitor server logs for unusual activity

## Troubleshooting

### Common Issues:

1. **"Module not found" errors:**
   - Ensure MCP SDK is installed: `npm install @modelcontextprotocol/sdk`
   - For Python: `python -m pip install mcp`

2. **Permission denied:**
   - Check file permissions on data directories
   - Ensure the user running MCP servers has access to stockroom data

3. **Connection refused:**
   - Verify server paths in configuration
   - Check that the stockroom dashboard is running
   - Ensure environment variables are set correctly

4. **Data not found:**
   - Verify STOCKROOM_DATA_DIR and STOCKROOM_FILES_DIR paths
   - Check that JSON data files exist in the data directory

### Logs and Debugging:

- Each MCP server logs to stderr
- Check your AI client's logs for connection issues
- Use the test commands above to verify server functionality

## Extending MCP Doors

To add new tools or servers:

1. Create a new server file in `mcp-servers/`
2. Implement the Server class with your custom tools
3. Add the server to `mcp-config.json`
4. Update client configurations

Example tool implementation:

```javascript
// In your server file
this.server.setRequestHandler('tools/call', async (request) => {
  const { name, arguments: args } = request.params;

  switch (name) {
    case 'my_custom_tool':
      return await this.myCustomTool(args);
    // ... other tools
  }
});
```
